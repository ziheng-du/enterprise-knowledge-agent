# 纯向量 vs Hybrid 检索对照

生成时间：2026-07-16 03:18 UTC

## 如何复现

```bash
python scripts/compare_retrieval_modes.py
python scripts/compare_retrieval_modes.py --output docs/retrieval_comparison.md
```

## 实验设置

- 对比对象：`vector`（纯向量 + 阈值）vs `hybrid`（向量 + BM25，经 RRF 融合）
- 不调用 LLM；直接调用检索器两路实现，避免 Agent 路由噪声
- 未套用文档密级 ACL（两路一致，聚焦召回差异）
- `top_k=4`，`score_threshold=0.3`（向量路过滤；与配置默认一致时见下）
- 当前配置默认：`RETRIEVAL_TOP_K=4`，`RETRIEVAL_SCORE_THRESHOLD=0.3`，`HYBRID_RRF_K=60`，`HYBRID_BM25_TOP_K=8`
- 向量库文档块数：44；Embedding / 切分策略以当前入库结果为准
- 判定：
  - **来源正确**：期望子串出现在命中文件名（如「报销」→「报销制度.md」）
  - **关键词命中**：top-1 正文包含约定关键片段（如「30天」「发票」）

## 解读（简历/面试可用）

- 分数列在两种模式下**不可直接横向比大小**：纯向量分为余弦相似度；Hybrid 展示分为 RRF 归一化与向量分的合成，主要用于排序，不是同一量纲。两路都命中且 BM25/向量 top-1 一致时，Hybrid 展示分常被归一化到约 1.0。
- 在小语料（本实验约 22 块）上，**来源级 Hit@K 容易打满**；更应看关键词是否进 top-1 正文，以及 Hybrid 是否因字面「30」等把无关制度拉进 Top-K。
- 若来源命中率持平，简历宜写「与纯向量对照实验，小语料下来源 Hit 持平，Hybrid 作专名/数字兜底默认策略」，**不要编造「提升 XX%」**。

## 结论

在本查询集（21 条）上，两路 **Top-1 来源**与 **Hit@4** 均为 21/21、21/21（持平）；Top-1 关键词命中为 纯向量 18/19、Hybrid 18/19。小语料下语义召回已较强，来源级指标触顶；Hybrid Top-4 平均独特来源数（3.05）高于纯向量（2.95），体现关键词路会拉入更多文档（含潜在干扰）。默认仍采用 `RETRIEVAL_MODE=hybrid` 作为专名/数字兜底，可用 `vector` 随时对照。
